🐧 Linux Permission Commands Every DevOps Engineer Should Know 🚀
------------------------------------------------------------------
Linux permissions are one of those topics that come up almost everywhere—daily work, troubleshooting, and especially DevOps interviews.
Instead of memorizing commands, try understanding when and why each command is used.

1.  umask – Sets the default permissions for newly created files and directories.
2.  chmod – Changes read, write, and execute permissions for files or directories.
3.  chown – Changes the owner of a file or directory (and optionally the group).
4.  chgrp – Changes only the group ownership of a file or directory.


💡 Quick Interview Tip:
-----------------------
1. ✅ umask → Default permissions for new files.
2. ✅ chmod → Modify file permissions.
3. ✅ chown → Change owner (and optionally group).
4. ✅ chgrp → Change only the group.


Understanding these four commands will help you confidently answer many Linux and DevOps interview questions.
